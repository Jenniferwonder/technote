Number 类型

  • ” uses the IEEE–754 format to represent both integers 整数 and floating-point values 浮点值 (also called double–precision values 双精度值 in some languages) ” Page 17

Number Literal Formats

  • ” Integers ” Page 17
  • ” octal literal ” Page 17
    • ” the first digit must be a zero (0) followed by a sequence of octal digits (numbers 0 through 7). ” Page 17
    • ” Octal literals are invalid when running in strict mode ” Page 17
  • ” hexadecimal literal ” Page 17
    • ” must make the first two characters 0x (case insensitive), followed by any number of hexadecimal digits (0 through 9, and A through F) ” Page 17
    • ” Numbers created using octal or hexadecimal format are treated as decimal numbers in all arithmetic operations. ” Page 17

” Floating-Point Values ” Page 17

  • To define a floating-point value, you must include a decimal point and at least one number after the decimal point. Page 17
  • ECMAScript always looks for ways to convert values into integers. Page 18
  • e-notation 科学计数法 Page 18
    • used to indicate a number that should be multiplied by 10 raised to a given power Page 18
      • let floatNum = 3.125e7; // equal to 31250000
      • 3e–17 Page 18
        • 0.00000000000000003 Page 18
    • ECMAScript converts any floating-point value with at least six zeros after the decimal point into e-notation Page 18
  • small rounding errors Page 18

    • Numbers are represented as 64 bits
